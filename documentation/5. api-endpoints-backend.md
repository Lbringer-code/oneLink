# Bundle API Specification

## Resources

- `/bundles`
- `/links` — not exposed directly; links are always accessed through their parent bundle.

---

## Endpoints

### POST `/bundles`

Create a new bundle.

**Request Body**

```json
{
  "title": "string",
  "links": [
    {
      "url": "string",
      "displayText": "string (optional)",
      "note": "string (optional)"
    }
  ]
}
```

**Success Response — `201 Created`**

```json
{
  "title": "string",
  "slug": "string"
}
```

**Error Response**

```json
{
  "error": "string"
}
```

---

### GET `/bundles/:slug`

Retrieve a bundle and all its links.

**Request**

No body required.

**Success Response — `200 OK`**

```json
{
  "slug": "string",
  "title": "string",
  "links": [
    {
      "url": "string",
      "displayText": "string (optional)",
      "note": "string (optional)"
    }
  ]
}
```

**Error Response**

```json
{
  "error": "string"
}
```

---

## Error Cases

| Scenario                              | Status Code       |
| ------------------------------------- | ----------------- |
| POST `/bundles` with zero links       | `400 Bad Request` |
| POST `/bundles` with malformed URL    | `400 Bad Request` |
| POST `/bundles` with missing title    | `400 Bad Request` |
| GET `/bundles/:slug` — slug not found | `404 Not Found`   |

---

## Background Jobs

**Stale bundle cleanup (cron):** Any bundle that has not been accessed in the last 30 days is automatically deleted.
