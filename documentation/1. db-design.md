# Database Design

## Domain Model

**Nouns:** Bundle, Link
**Verbs:** Bundle contains Links. Bundle expires (via cleanup). Bundle is accessed (updates `last_accessed`).

---

## Entities and Attributes

### Bundle

| Attribute       | Type      | Constraints              |
|-----------------|-----------|--------------------------|
| `slug`          | nanoid    | **PK**, unique           |
| `title`         | text      | NOT NULL                 |
| `created_at`    | timestamp |                          |
| `last_accessed` | timestamp | NOT NULL                 |

### Link

| Attribute      | Type      | Constraints                          |
|----------------|-----------|--------------------------------------|
| `bundle_slug`  | text      | NOT NULL, **FK** → `bundle(slug)` ON DELETE CASCADE |
| `url`          | text      | NOT NULL                             |
| `display_text` | text      |                                      |
| `note`         | text      |                                      |
| `created_at`   | timestamp |                                      |

**Composite PK:** `(bundle_slug, url)` — prevents duplicate URLs within the same bundle.

---

## Relationships

```
Bundle 1 ──── N Link
```

- One Bundle contains many Links (1:N).
- Link belongs to Bundle via `bundle_slug`.
- CASCADE delete: removing a Bundle removes all its Links.

---

## Indexes

| Index                      | Column(s)        | Purpose                        |
|----------------------------|------------------|--------------------------------|
| PK (automatic)             | `bundle.slug`    | Bundle lookup by slug          |
| `idx_bundle_last_accessed` | `last_accessed`  | Cleanup query performance      |
| CPK (automatic)            | `(bundle_slug, url)` | Link uniqueness + bundle fetch |

---

## Access Patterns

| Pattern                          | Mechanism                        |
|----------------------------------|----------------------------------|
| Lookup bundle by slug            | PK scan on `bundle.slug`         |
| Fetch all links for a bundle     | Index scan on `bundle_slug` (CPK)|
| Delete stale bundles             | Index scan on `last_accessed`    |

---

## Cleanup Strategy

- No explicit expiry column.
- Bundles not accessed within a configurable threshold (e.g. 30 days) are deleted.
- `ON DELETE CASCADE` handles link cleanup automatically.

```sql
DELETE FROM bundle
WHERE last_accessed < NOW() - INTERVAL '30 days';
```
